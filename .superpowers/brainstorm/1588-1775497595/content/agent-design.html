Autonomous Agent Design

Claude-powered analyst that monitors, reasons, and publishes

Agent Loop

Execution cycle (every 4 hours via Vercel Cron)
1. FETCH — Pull latest NASA NEO data + run rule-based enrichment
2. RECALL — Load agent memory from Vercel KV (past observations, tracked objects, trend history)
3. ANALYZE — Send data + memory to Claude with analysis prompt:
"Compare current NEO data against your previous observations.
 Identify: new objects, orbital revisions, anomalous velocities/sizes,
 objects crossing alert thresholds, emerging trends."
4. DECIDE — Claude determines actions via tool calls:
• Flag objects for monitoring
• Generate scene annotations
• Publish threat assessment (if warranted)
• Send email alert (if critical)
• Update daily briefing
5. PERSIST — Save updated memory to KV (what it observed, what it decided, why)

Agent Tools (Claude)

save_observation

Record what the agent noticed. Params: object_id, observation_type (new_object, revision, anomaly, trend), details, severity. Written to KV with timestamp.

annotate_scene

Write a 3D annotation for the dashboard. Params: object_id, label, severity, explanation, priority. Stored in KV, rendered by CSS2DRenderer in the scene.

publish_threat

Create a public threat assessment page. Params: object_id, title, analysis (markdown), risk_level, comparison_objects[]. Writes to a dynamic route /threats/[id].

update_briefing

Generate or update the daily briefing. Params: summary, highlights[], watchlist[], stats. Served at /briefings and shown as dashboard widget.

send_alert

Email notification for critical events. Params: subject, body (markdown), urgency. Uses existing Resend integration. Agent decides when to alert — no fixed thresholds.

compare_historical

Look up past observations for an object or class. Params: object_id or query (e.g. "fast approaches in last 30 days"). Reads from KV memory. Gives the agent temporal context.

Agent Memory (Vercel KV)

KV key structure
agent:observations:{date} — array of observations from that run
agent:watchlist — objects the agent is actively tracking (with reasons)
agent:annotations — current scene annotations (read by dashboard)
agent:briefing:latest — most recent daily briefing
agent:briefing:{date} — archived briefings
agent:threats:{object_id} — published threat assessments
agent:memory:summary — rolling context window (last 30 days condensed)
TTL: observations expire after 90 days. Briefings after 30 days. Annotations after 48 hours (agent refreshes each run).

Replaces Current Monitoring

The current /api/monitoring cron is threshold-based: it checks if rarity > threshold and sends an email. The new agent replaces this entirely. Instead of fixed rules, Claude reasons about whether something is notable, compares against its memory of past observations, and decides what actions to take. The email alert becomes one of several possible outputs, not the only one.
