Visual Upgrade Design

From solid Three.js to cinematic portfolio piece

Layer 1: Post-Processing Pipeline

Using the postprocessing library you already have installed but aren't using

Effect Stack (applied in order)
Bloom Sun corona bleeds light realistically. High-risk asteroids glow warm. Stars twinkle with soft halos.
Tone Mapping ACES Filmic tone mapping for cinematic HDR look. Bright sun doesn't wash out dark space.
Vignette Subtle darkening at edges. Draws focus to the center of the scene. Cinematic framing.
Chromatic Aberration Very subtle (0.002). Adds photographic realism at edges. Disabled on mobile.
God Rays Volumetric light from the Sun. Visible when Sun is partially occluded by Earth/Moon.

Layer 2: Asteroid Visual Overhaul

From colored spheres to recognizable space rocks

Current: Colored Spheres
Upgraded: Procedural Asteroids
Approach: Procedurally deformed IcosahedronGeometry with Simplex noise displacement. 3 LOD levels (high detail when selected, medium nearby, low distant). Custom PBR shader with roughness variation for rocky surface. Risk level shown via emissive glow ring, not sphere color.

Layer 3: Particle Effects

Solar Wind

Instanced particle stream from Sun. Thousands of points flowing outward with velocity falloff. Subtle, always-on ambient effect.

Asteroid Trails

Fading trail behind each asteroid showing recent trajectory. Length proportional to velocity. High-risk asteroids get brighter trails.

Dust & Debris

Sparse micro-particles around asteroid belt region. Adds depth and scale to the scene. GPU instanced for performance.

Layer 4: Cinematic Camera

Close Approach Mode

When an asteroid is selected, camera smoothly transitions to a dramatic tracking shot — following the asteroid as it sweeps past Earth. Depth of field blur on background. Triggered by user or by AI chat command.

AI Director Mode

The chat AI can choreograph camera movements: "Show me this asteroid's approach" triggers a scripted flyby sequence. Camera paths are defined as tool outputs from Qwen.

Layer 5: AI Scene Annotations

The standout feature — AI-generated HUD overlays in 3D space

AI Analysis Overlay (concept)
2024 BX1 — HIGH RISK
Unusually fast approach (24.3 km/s). Miss distance revised closer by 12% since yesterday. Agent flagged for monitoring.
Annotations by Claude Agent • Updated 2h ago
How it works: The autonomous Claude agent generates structured annotation data (position, label, severity, explanation) stored in Vercel KV. The 3D scene renders these as HTML overlays anchored to world-space positions using Three.js CSS2DRenderer. Annotations update when the agent runs, not on every frame — zero performance cost.
