# AstroWatch — Full Application Review (2026-07-03)

Scope: entire app (`astro-watch/`), reviewed on `main` @ `ab47046` across five dimensions:
correctness bugs, security, architecture/performance, visual design/UX (code-level), and a
hands-on inspection of the live site (www.astro-watch.com, desktop 1440px).
Paths are relative to `astro-watch/` unless noted.

**Overall verdict:** The foundations are genuinely good — correct impact physics with honest
documentation, clean server/client split, careful rate limiting, and the April 2026 security
priorities (CRON_SECRET, CORS) are fixed. The problems cluster at *integration seams*: the
agent→UI annotation contract is dead, units are inconsistent (m vs km, degrees vs radians,
AU vs scene units), rarity colors diverge across ~6 modules, and one 2,247-line component
drives most of the render-performance problems. Nothing here is fatal; most of the
high-impact fixes are small.

## 1. Correctness bugs

1. **[High] Agent scene annotations never render — client schema doesn't match stored data**
   `components/visualization/3d/AgentAnnotations.tsx:7-14,41,49,52` vs `lib/agent/memory.ts:29-35`
   Component expects `{objectId, severity, explanation}`; `/api/agent-data` returns
   `{asteroidId, label, color, notes}`. `ann.objectId` is always undefined → every annotation
   renders `null`. The agent's `annotate_scene` feature is invisible in production.
   *Fix:* change the component to consume `SceneAnnotation` (`asteroidId`, `label`, `color`, `notes`).

2. **[High] Even after #1, annotations are positioned in the wrong coordinate space**
   `AgentAnnotations.tsx:52`, `lib/nasa-api.ts:260-274`, `EnhancedSolarSystem.tsx:1210-1217,1777-1789`
   `asteroid.position` is raw AU with its own random angle; meshes actually render at
   `orbit.phase/radius` (64 units/AU) inside the moving Earth group. Labels would float near the
   Sun, detached from asteroids. `calculatePosition()` is effectively dead/incorrect code.
   *Fix:* compute label position from the same `orbit.phase/radius` math (or store rendered
   positions); delete `calculatePosition`.

3. **[High] Lost-update race in agent memory (parallel tools + read-modify-write)**
   `lib/agent/run.ts:200-214`, `lib/agent/tools.ts:206-246`
   Tool calls in one turn run via `Promise.all`; `save_observation`/`annotate_scene` each do
   loadMemory → mutate → save whole array. Two saves in one turn silently drop one.
   *Fix:* execute tool calls sequentially, or use per-asteroid KV keys.

4. **[Medium] One asteroid with empty `close_approach_data` 500s the whole feed**
   `lib/nasa-api.ts:163-165,232,264,278-279` — unguarded `close_approach_data[0]` in four places
   rejects the `Promise.all` at line 146 → entire `/api/asteroids` fails.
   *Fix:* filter such objects out before enrichment.

5. **[Medium] `/api/monitoring` `maxDuration=60` too short for the agent loop**
   `app/api/monitoring/route.ts:4`, `lib/agent/run.ts:41,155-219` — up to 10 sequential Claude
   calls + up to 3 Opus advisor calls will exceed 60s; Vercel kills mid-run leaving memory
   partially written. *Fix:* raise maxDuration (Pro allows 300s) and add an internal wall-clock
   budget that forces a final persist.

6. **[Medium] APOD fallback can CDN-cache the wrong day's picture**
   `lib/nasa-api.ts:106-122`, `app/api/apod/route.ts:11-18` — any failure (incl. transient) falls
   back to `date-1`, then cached under the requested date for up to 25h.
   *Fix:* only fall back when no explicit date was requested; skip fallback on 4xx.

7. **[Medium] Chat tool loop can end with no assistant text**
   `app/api/chat/route.ts:195-306` — if the model still calls tools on the 3rd iteration, tools
   execute but no final LLM call is made; client gets `done` with a blank message.
   *Fix:* final iteration with `tool_choice: 'none'` to force a text answer.

8. **[Medium] `orbit.inclination` unit mismatch: NASA degrees vs ~radian fallback**
   `lib/nasa-api.ts:237` (consumer `EnhancedSolarSystem.tsx:1217`) — real-data asteroids bob with
   ~200× the amplitude of fallback ones. The `any` return type (line 230) hides this.
   *Fix:* normalize units at ingest; type the return.

9. **[Medium] Rate limiter can permanently block an IP**
   `lib/rate-limit.ts:41-49` — non-atomic `incr`/`expire`: a crash between them leaves a key with
   no TTL → permanent 429 for that IP. *Fix:* atomic pattern, or set expire whenever `ttl < 0`.

10. **[Low] Chat route 500s on non-string `content`** — `app/api/chat/route.ts:147-161`; also
    client-supplied `tool_calls` survive sanitization. *Fix:* type-check content, strip tool fields.
11. **[Low] Mock data unit bugs (dev-only)** — `lib/nasa-api.ts:323,353-354,369,375`: size stored
    in km in a meters field; orbit speed ~1000× too fast.
12. **[Low] Risk score uses max diameter while display uses mean** — `lib/nasa-api.ts:162` vs `277`.
13. **[Low] `computeApproachCurve` fractional `setDate` truncation skews date labels** —
    `lib/approach-curve.ts:29`. *Fix:* `new Date(closest.getTime() + days*86400000)`.
14. **[Low] `getAPOD` fallback date is timezone-dependent** — `lib/nasa-api.ts:111-114`; append `Z`.
15. **[Low] Threat assessments never expire** — `lib/agent/memory.ts:173-175` (only KV artifact
    with no TTL); `/threats/[id]` can serve months-old assessments.
16. **[Low] `inferAsteroidType` labels 'S' but returns DEFAULT density; M-type unreachable** —
    `lib/impact-physics.ts:54-68`.
17. **[Low] Duplicated "today+7d" fetch logic in 3 places; "sliding window" comment is wrong** —
    `app/api/asteroids/route.ts:9-29`, `app/api/chat/route.ts:27-31,84`, `lib/agent/run.ts:120-127`.

## 2. Security

**Previously flagged (April 2026): status**
- CRON_SECRET on `/api/monitoring`: **fixed** (`app/api/monitoring/route.ts:6-17`, fails closed).
- CORS wildcard: **fixed** (`middleware.ts:4-40`, allowlist only).
- Content filter / tool constraints: **partially fixed** — all chat tools are read-only (good),
  but see S3 below.
- Rate limiting: **largely fixed** — KV-backed + global daily budget; residuals in #9 above and
  fail-open on KV error (deliberate).

18. **[High] Outdated Next.js 15.3.6 + 14 vulnerable deps (1 critical, 8 high)**
    `package.json` — image-optimization CVEs (relevant: remote images allowed), middleware
    redirect SSRF, RSC DoS; also `ws`, `js-cookie` (via react-use), `@anthropic-ai/sdk` 0.82.0.
    *Fix:* upgrade Next 15.x latest patch, `npm audit fix`, bump SDK. Highest-leverage security fix.

19. **[Medium] Agent `send_alert` recipient is model-controlled**
    `lib/agent/tools.ts:154-158,183,280-284` — required `to` param takes precedence over
    `ALERT_TO_EMAIL`; the model can't read env vars so it fabricates/copies an address; persisted
    memory (prior model output) is an injection loop into this. Also a correctness bug.
    *Fix:* remove `to` from the schema; hardcode `ALERT_TO_EMAIL` server-side, fail if unset.

20. **[Medium] `.env.example` drift breaks alert recipient + leaks a personal address**
    `.env.example:15-22` documents `NOTIFICATION_EMAIL` etc. — none read by code (code reads
    `ALERT_TO_EMAIL`); line 17 commits `0xFriday@pm.me`. *Fix:* replace stale keys, delete address.

21. **[Medium] HTML injection into alert emails from model-controlled strings**
    `lib/email.ts:35,37-50,74` — `a.name`, dates, `subject` interpolated unescaped into email HTML.
    *Fix:* HTML-escape all interpolated strings; fixed subject format.

22. **[Medium] Chat filter/length check only inspects the LAST user message**
    `app/api/chat/route.ts:147-161,184` — earlier user messages forwarded unfiltered and
    unbounded (injection bypass + token-cost abuse). *Fix:* validate every user message, cap
    total payload.

23. **[Low] `/api/apod` `date` unvalidated → query-string riding on server's NASA key**
    `app/api/apod/route.ts:7-11`, `lib/nasa-api.ts:88-90`. *Fix:* enforce `^\d{4}-\d{2}-\d{2}$`.
24. **[Low] Dead open proxy rewrite `/api/nasa/(.*) → api.nasa.gov` in `vercel.json`** — unused;
    delete it.
25. **[Low] `/threats/[id]` KV read with unvalidated ID** — `app/threats/[id]/page.tsx:11-12`;
    add `/^\d+$/` check.

Verified clean: no secret leakage to client bundles, no committed `.env` in history, no SSRF
(fixed hosts), chat cannot reach agent write-tools.

## 3. Architecture & performance

26. **[High] Hover state at page level re-renders the entire 3D scene per pointer event**
    `app/dashboard/page.tsx:28,48,112-118`, `lib/store.ts:53-66`,
    `EnhancedSolarSystem.tsx:1195-1274,2078-2137` — plus `getFilteredAsteroids()` returns a new
    array per call, invalidating trail `useMemo`s (Float32Arrays, CatmullRom curves, tube
    geometries rebuilt per hover). Frame drops just from moving the mouse.
    *Fix:* hover in Zustand or a ref + imperative material update; `React.memo` asteroid/list
    rows; memoize filtered array on `[asteroids, riskFilter]`.

27. **[High] `ProceduralAsteroid` rebuilds geometry on every hover, never disposes**
    `ProceduralAsteroid.tsx:63-68` — LOD `detail` depends on hover/selection → new
    IcosahedronGeometry + CPU displacement + normals per hover in/out; old geometry leaks.
    *Fix:* precompute LODs once per seed; dispose replaced geometry.

28. **[High] No instancing: 400–700 draw calls (3+ per asteroid, 104 orbit-decoration meshes)**
    `EnhancedSolarSystem.tsx:1209-1253,1359-1373,416-469`. *Fix:* InstancedMesh for asteroids,
    merged geometry for trails and the 96 marker spheres.

29. **[High] Star twinkle: CPU loop over 8,000 stars + 32KB attribute upload every frame**
    `EnhancedSolarSystem.tsx:1112-1130`. *Fix:* shader-based twinkle (time uniform + static
    phase attribute).

30. **[High] Single client bundle: all views statically imported; dead postprocessing import**
    `app/dashboard/page.tsx:6-15`, `EnhancedSolarSystem.tsx:13` — three.js + recharts +
    postprocessing (+~300KB gz) ship on first paint regardless of view.
    *Fix:* `next/dynamic` per viewMode + ChatPanel/ImpactSimulation; delete the
    `PostProcessingEffects` import.

31. **[High] Root layout is `'use client'` — no server rendering anywhere**
    `app/layout.tsx:1` — landing page ships as JS; no `metadata` export (hand-written head tags).
    *Fix:* server layout + small `Providers.tsx` client component; use `metadata`.

32. **[Medium] Chat scene-control writes are read by nobody**
    `lib/store.ts:6,11,43,48`, `ChatPanel.tsx:20,38`, `app/dashboard/page.tsx:27` — dashboard
    keeps `selectedAsteroid` in local useState; chat writes the store copy; `cinematicTarget`
    has zero readers. The chat "select asteroid" command silently does nothing.
    *Fix:* store as single owner; delete local state; implement or remove `cinematicTarget`.

33. **[Medium] `Math.random()` in enrichment: scene reshuffles every 15-min refetch**
    `lib/nasa-api.ts:237-238,251-252,262` — every visitor sees a different scene; every refetch
    jumps all asteroids. *Fix:* seed phase/speed from a hash of `asteroid.id`.

34. **[Medium] `/api/asteroids` over-fetches and has no CDN cache header; `range=month` lies**
    `app/api/asteroids/route.ts:19-23,31-36` — spreads full raw NASA objects (hundreds of KB);
    no `s-maxage`; month silently returns 7 days. *Fix:* trimmed DTO, `s-maxage=300 swr=3600`,
    chain 7-day calls or drop the month option.

35. **[Medium] God component: `EnhancedSolarSystem.tsx` (2,247 lines, ~15 components)**
    Textures, shaders, planets, star field, asteroid field, trails, camera UI, info panel,
    desktop sidebar + near-duplicate mobile drawer. Hides bugs #36/#37.
    *Fix:* split into textures/shaders/Earth/Sun/Moon/StarField/AsteroidField/CameraControls/
    AsteroidListPanel modules.

36. **[Medium] Mobile list FAB shares state with detail modal**
    `EnhancedSolarSystem.tsx:2047-2053,2162-2171,2174-2242` — `showDetailedView` doubles as
    modal flag and drawer toggle: opening the mobile list with a selection also opens the modal.
    *Fix:* separate `showMobileList` state.
37. **[Medium] Mobile drawer hover compare: list index vs `parseInt(asteroid.id)` — never matches**
    `EnhancedSolarSystem.tsx:2202`.
38. **[Medium] Earth over-tessellated (256×256 ≈ 131k tris) + 4096×2048 procedural fallback
    texture painted synchronously on every mount, never disposed**
    `EnhancedSolarSystem.tsx:475,536-694,791,808,821-846`. *Fix:* 64×64 segments; lazy fallback
    only on load error; dispose replaced textures.
39. **[Low] Per-asteroid dynamic imports in the enrichment hot path** — `lib/nasa-api.ts:176,215`;
    hoist to static imports.
40. **[Low] Unused heavy deps (`d3`, `leva`, `react-spring`, `@react-spring/three`) + tracked
    `.backup` file** — `package.json`, `InteractiveGlobe.tsx.backup`; remove.
41. **[Low] `RiskDashboard` recomputes all chart datasets every render (no `useMemo`)** —
    `RiskDashboard.tsx:21+`.

## 4. Visual design & UX

### Data honesty (the project's stated identity)

42. **[High] Rarity colors diverge across ~6 modules — same level, different hue per view**
    `RiskLegend.tsx:14-71,191-217`, `RiskDashboard.tsx:102-106,338-361`,
    `TrajectoryAnalysis.tsx:133-137`, `DetailedAsteroidView.tsx:13-19`, `ImpactRiskMap.tsx:229-240`
    R2 is green in the legend, amber in pie bins, neon green in 3D, yellow in the map; the 3D
    ramp ends in magenta while the legend teaches red.
    *Fix:* one exported `RARITY_COLORS` scale (blue→green→yellow→orange→red) consumed everywhere.
    **The single highest coherence-per-line change in the app.**

43. **[High] `risk` shown as "AI-calculated probability %" with an ML claim**
    `DetailedAsteroidView.tsx:229-258,358` — it's a rule-based display score, there is no ML.
    Contradicts the honest-copy principle. *Fix:* rename "Severity index", drop `%`, rewrite
    tooltips.

44. **[High] Fabricated "Real-Time Monitoring" telemetry**
    `MonitoringDashboard.tsx:78-91,106,111-136,228-229` — invented discovery alerts, simulated
    "new this week", fake observatory statuses/timestamps, hardcoded "OPERATIONAL".
    *Fix:* remove the tab or badge every block "illustrative — not live telemetry".

45. **[High] Size labeled km where the value is meters (3 places)**
    `RiskDashboard.tsx:199,321,332`, `MonitoringDashboard.tsx:220` — confirmed live:
    dashboard shows "Average size: 230.6 km diameter".

46. **[Medium] Fake status chips** — `RiskDashboard.tsx:398-444` hardcoded green "NASA API
    Status • Live"; `app/dashboard/page.tsx:184` "Last Updated" is render time, not fetch time.
    *Fix:* drive from TanStack Query `dataUpdatedAt`/`isError`.
47. **[Medium] Orbital classification derived from the *visualization* radius, buckets overlap**
    `TrajectoryAnalysis.tsx:20-34` — use `orbital_data` semi-major axis/perihelion or relabel
    "approximate".
48. **[Medium] PHA flag has no visual encoding in the 3D scene** — the one NASA-official hazard
    signal is text-only. *Opportunity:* dashed ring or slow pulse on PHA objects.

56. **[Low] Unformatted large stats** — "807547.9 PJ", "193004.0 megatons", "×33763.8"; add

57. **[Low] Everything is "R0 - Routine" on a typical day** ("Rare: 0" in the footer) — the list

58. **[Low] Landing hero fade-in is slow** (~4s before text is readable).
